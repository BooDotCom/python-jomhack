.gitignore: when sharing the files in a repository, stuff
that is written in this file will be ignored

certain files act like the keys to your house(repository), hence
adding them into .gitignore prevents them from being shared.

__pycache__: acts as a save point. instead going from start to finish,
the code goes straight to the save point.

the 3 commands of adding changes to a github repo:
git add .
git commit -m "yer cmment"
git push

running a file(python for now) in terminal:
python "(full file name, plus its extension)"

add indent: tab
remove indent: shift + tab

apparently you CAN install multiple packages(?) at once. eg:

line above installs 3 packages at once.

so yeah, you don't want your device to commit sudoku, don't cha?
